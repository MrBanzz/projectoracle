## Goal

Build a Next.js + React Flow MVP dashboard that takes a GitHub repository URL, generates an architecture map and dependency graph from mock data, and produces an impact analysis for any selected module — answering the headline question "What breaks if this module changes?" — with a dark, cyberpunk-accented developer aesthetic.

## Context

- The workspace `/home/harness/cyops_data/workspace/mvp` is empty except for `.gitignore` and `.humanize/`. This is a greenfield MVP.
- Target experience: a single-page developer dashboard, not a multi-route product. The end-to-end workflow is linear: URL → Analyze → Architecture Map → Dependency Graph → Select Module → Impact Analysis. A wizard or vertically stacked sections are both viable; a single dashboard with progressive disclosure (analysis unlocks the lower panels) is the cleanest.
- The product is a hackathon demo. No production concerns (auth, billing, persistence, RLS, rate limits, telemetry, real GitHub auth) are required. Mock data is explicitly allowed.
- The visual reference set (GitHub Insights, Linear, Vercel) implies: dense information layout, monospaced metadata, restrained color, panel-based sections. "Cyberpunk accents" implies: a strict dark base (near-black) with neon cyan/magenta/violet highlights on key states (selection, focus, risk indicators), thin glowing borders, and small HUD-style labels — used sparingly, not everywhere.
- Tech stack is fixed: Next.js (App Router), TypeScript, Tailwind CSS, React Flow. The plan must use only these.
- No external API integrations. No auth. No backend persistence. A static or client-side mock dataset is sufficient.
- Reasonable sample repos: a fictional monorepo (e.g., "acme/payments-platform") is preferable to cloning a real OSS repo because the mock graph can be hand-tuned to look balanced in React Flow and to contain obvious high-risk coupling for the impact analysis to demonstrate.

## Assumptions

- The implementer may use Next.js 14+ with the App Router and React 18+; the latest stable Next.js and React Flow (`reactflow` v11) are acceptable.
- The implementer may install dependencies freely (no offline constraint). Standard `npm` or `pnpm` is fine; `pnpm` is preferred for speed.
- All data is mocked statically under `lib/mock/`. No network calls at runtime.
- The dashboard is single-page. URL state (e.g., `?repo=acme/payments-platform&module=checkout`) drives the selected repository and selected module, making the demo shareable and reload-safe.
- The plan targets desktop (≥1280px) primarily; layouts should remain usable down to ~1024px and not break below that, but mobile is not a goal.
- "Cyberpunk accents" is interpreted as restrained neon highlights on a near-black surface, not a heavy sci-fi theme. No animated grid backgrounds or glitch effects in the primary flow; a subtle scanline or corner-bracket motif on key cards is acceptable.
- TypeScript is strict mode; ESLint and Prettier default configs are sufficient.
- No tests are required for the MVP beyond a smoke build (`next build`) and a manual checklist verification.
- The `pnpm-lock.yaml` / `package-lock.json` is committed; no monorepo tooling.

- **Styling tokens** (define in `tailwind.config.ts` and reference via `bg-surface-base`, `text-accent-cyan`, etc.):
  - `surface.base` = `#0a0a0f`
  - `surface.panel` = `#111118`
  - `surface.raised` = `#16161f`
  - `border.subtle` = `#1f1f2b`
  - `text.primary` = `#e6e6f0`
  - `text.muted` = `#8a8a99`
  - `accent.cyan` = `#22d3ee` (used for: active nav, focus ring, selected node ring, primary action)
  - `accent.magenta` = `#f0abfc` (used for: CRITICAL risk, high-severity debt)
  - `accent.violet` = `#a78bfa` (used for: HIGH risk, secondary highlights)
  - Risk colors: LOW = `#22d3ee` (cyan), MEDIUM = `#a78bfa` (violet), HIGH = `#f0abfc` (magenta), CRITICAL = `#fb7185` (rose-400)
- **Mock data shape** (minimal — extend as needed):
  - `RepoData`: `{ id, fullName, description, languages: { name, pct }[], totals: { files, modules }, health: { score, complexity, debtIndex, coverage, deltas: { score, complexity, debtIndex, coverage } }, analyzedAt }`
  - `ArchGraph`: `{ nodes: ArchNode[], edges: { from, to, kind }[] }`
  - `DepGraph`: `{ nodes: DepNode[], edges: { from, to }[] }` where `DepNode = { id, label, files, inCount, outCount, layer }`
  - `ImpactReport`: `{ moduleId, risk: 'LOW'|'MEDIUM'|'HIGH'|'CRITICAL', direct: string[], transitive: string[], recommendations: string[] }`
  - `DebtItem`: `{ id, title, category, severity, description }`
- **State sharing**: a small hook `useRepoSelection()` reads `searchParams` and exposes `{ repoId, moduleId, setRepo, setModule }`. Components read from it; do not duplicate state in component-local `useState`.
- **Graph layout**: hand-lay node positions in the mock data (`x`, `y` per node). This avoids pulling in `dagre` or `elkjs` and keeps the demo deterministic. Use React Flow's `fitView` on mount.
- **Module selection highlight**: pass a `selected` boolean into `DepNode`; when true, render a 2px cyan ring (Tailwind `ring-2 ring-accent-cyan`). Same module in `ArchNode` should show a subtle ring too for consistency.
- **Empty states** (must exist, not throw):
  - Unknown repo → centered card: "Repository not in demo catalog" + "Try a sample repo" buttons for the two seeded ids + link to clear `?repo=`.
  - No module selected → Impact Analysis shows a dashed border placeholder with "Select a module to see what breaks." with a hint pointing to the Dependency Graph.
- **Performance**: React Flow is the only heavy dep. Lazy-import `reactflow` inside the panels with `next/dynamic({ ssr: false })` to keep first paint fast.
- **No backend**: do not create API routes. All mock data is imported directly from `lib/mock/`.
